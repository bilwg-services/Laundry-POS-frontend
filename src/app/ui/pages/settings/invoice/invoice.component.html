Invoice Settings

Automatically set the invoice prefix and starting number of the invoice. If the invoice prefix is FABKLEAN- and the start number is 1000, the first invoice ID generated will be FABKLEAN-1001. If there are already 34 invoices created, the next order will generate FABKLEAN-1035.

Invoice# generation can be done at different stages of the order workflow. It all depends on your business model. If you don't want to pay taxes to the government for unpaid orders, set this to "Paid."

This can be done either by not applying tax at the order level from POS, or if all items in the order are of type zero tax.
